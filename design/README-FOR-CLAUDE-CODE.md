## مفاتيح القراءة للتطوير
1. **الشريط الداكن أعلى كل صفحة (pkmeta) أداة مراجعة — ليس UI منتج.** أزراره تبدّل «الشاشات المنطقية» (data-view) و«حالات الشاشة» (data-state). عند التطوير: كل data-view = شاشة/حالة توجيه، كل data-state = حالة UI.
2. **الوسم «مرحلة 2 — مبنية»**: واجهات مبنية كاملة لكن أولوية تنفيذها بعد الطيار (docs/20 + docs/21 §3). ابنِ جداولها وعقودها من اليوم الأول.
3. **أسماء حالات الطلب السبع للعميل حرفية** (PRD §7) — لا تعدلها.
4. **الخصوصية:** اللوحة تظهر كاملة فقط لموظف الفرع في الطلب النشط؛ عدا ذلك مقنّعة (•••• 8342 / 05X *** XX21).
5. **الأرقام لاتينية دائماً** في اللوحات والرموز والمبالغ (IBM Plex Mono).
6. الروابط عميقة: `customer/P7.html#C-45` يفتح حالة بعينها — استخدمها في تذاكر التطوير.
