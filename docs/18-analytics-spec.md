# 18 — مواصفة التحليلات (Analytics Spec)

المصدر السابق: 12 · المصدر: analytics_events (من الأحداث والواجهات) + استعلامات تشغيلية

---

## 1. رحلة العميل (قمع)

مشاهدة المطعم → فتح القائمة → إضافة للسلة → بدء الدفع → نجاح الدفع · تكرار الطلب · التخلي عن السلة (وأين) · أسباب فشل الدفع.

## 2. التشغيل (قلب المنتج)

| المؤشر | الملاحظة |
|--------|----------|
| زمن قبول الطلب | مقابل عداد BR-1 |
| التجهيز المتوقع مقابل الفعلي | دقة وعد الوقت |
| **نسبة جاهزية الطلب قبل وصول العميل** | المؤشر الأول للمنتج |
| زمن انتظار العميل بعد الوصول | مقابل زمن الخدمة المستهدف |
| زمن خروج الموظف · زمن التسليم | |
| نسبة اكتشاف الاقتراب آلياً | صحة الطبقات 2–3 |
| نسبة استخدام زر «وصلت» اليدوي | مؤشر صحة الجيوفنس |
| نسبة بيانات السيارة غير الصحيحة | جودة الملفات |
| نسبة No-show | BR-3 |

## 3. التاجر

معدل قبول ورفض الطلبات · نسبة إغلاق استقبال الطلبات (وضع الازدحام) · معدل الإلغاء · تقييم الفروع · الفروع المتصلة · دقة القائمة والتوفر.

## 4. المالية

GMV · الإيراد لكل طلب · رسوم الدفع · تكلفة الإشعارات والخرائط · تكلفة الدعم لكل طلب · الاسترجاعات · صافي هامش المساهمة · LTV · CAC.

## 5. التقنية

API P95 · نسبة أخطاء الخادم · تأخر WebSocket · تأخر Push · فشل Webhooks · الوظائف المتوقفة · المدفوعات المكررة (يجب صفر) · الأحداث المفقودة (Outbox مقابل المستهلَك) · Crash-free sessions.

## 6. تصنيف الأحداث

snake_case، مصدرها الأساسي أحداث النطاق (وثيقة 12) + أحداث واجهة للقمع (`store_viewed`, `menu_opened`, `item_added`, `checkout_started`, `payment_succeeded/failed`, `cart_abandoned`, `trip_started`, `arrived_confirmed`, `manual_arrival_used`, `handoff_verified`, `review_submitted`, `ticket_opened`…). خصائص قياسية: merchant_id, branch_id, order_id, city, platform, session_id — **بلا بيانات شخصية** (وثيقة 17).

## 7. اللوحات

1. **لوحة الطيار اليومية:** §2 كاملاً + القمع + No-show.
2. **لوحة الموثوقية:** §5 + فشل الإشعارات + طوابير DLQ.
3. **لوحة التاجر داخل المنتج** (M-02/M-14): من الاستعلامات التشغيلية مباشرة.
4. **لوحة الأدمن** (A-01): GMV والنمو والاحتفاظ والمدن.

## 8. قواعد

المقاييس التشغيلية من order_status_history وpickup_* حصراً (لا اعتماد على العميل) · مراجعة أسبوعية أثناء الطيار تُدوَّن قراراتها · كل مؤشر له مالك وهدف يُثبَّت بعد أول 4 أسابيع بيانات حقيقية.
