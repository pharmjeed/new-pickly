## 3. قواعد الإنفاذ

1. الصلاحيات تُحمل في الجلسة وتُفحص في API، والعزل يُنفذ في Repository — طبقتان دائماً.
2. بيانات حساسة (لوحة كاملة، جوال كامل، رحلة العميل) مشروطة بالطلب النشط + الفرع المعني + الدور.
3. MFA إلزامي: مالك التاجر، المالية، وكل أدوار Pickly.
4. كل فعل إداري حساس (استرجاع، تعليق، تعديل حالة، تجاوز عزل، تغيير بنكي) → audit_logs (قبل/بعد + سبب).
5. أي endpoint جديد يُضاف سطره هنا قبل التنفيذ.
