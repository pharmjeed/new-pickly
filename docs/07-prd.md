# 07 — متطلبات المنتج (PRD)

المصدر السابق: 01–06 · المتطلبات مرقمة وقابلة للتتبع إلى الشاشات (04) والحالات (05) والقواعد (06)

---

## 1. تطبيق العميل

- **FR-C01** تصفح كزائر؛ التسجيل جوال+OTP عند الدفع (BR-13). شاشات C-01–C-08.
- **FR-C02** الاكتشاف: قريبة/الأسرع/عروض/سبق طلبت/الأكثر طلباً + بحث واقتراحات + تصنيفات + خريطة بزمن وصول + فلاتر C-16.
- **FR-C03** صفحة المطعم بفرع مختار + **تنبيه الفرع الأقرب** + تغيير الفرع يعيد فحص التوفر والأسعار (C-19/C-20).
- **FR-C04** منتج بتخصيص كامل (C-24/C-25) مع منع الإضافة قبل الإلزامي.
- **FR-C05** سلة بتنبيه زمني ذكي (وصولك X وتجهيز Y) + معالجة تغير التوفر (C-26/C-27).
- **FR-C06** وقت الاستلام: ASAP / مجدول بفترات وسعة (BR-5) / «سأتحرك لاحقاً».
- **FR-C07** سيارات متعددة (C-29/C-30) بلوحة مشفرة ومقنّعة، نقطة استلام إن تعددت، تعليمات الفرع (C-31).
- **FR-C08** كوبونات بسبب رفض واضح (BR-7).
- **FR-C09** دفع: Apple Pay/مدى/بطاقات/محافظ البوابة، Tokenization فقط، 3DS، منع تكرار (C-33–C-36، وثيقة 13).
- **FR-C10** الطلب النشط (C-38–C-52): شريط 7 حالات، انتظار قبول بعداد، «أنا في الطريق»، رحلة بخريطة وETA، «وصلت» يدوي دائماً، موقف، تأكيد سيارة، طابور، رمز استلام، «لا أرى الموظف».
- **FR-C11** بطاقة الطلب النشط العائمة أسفل كل الشاشات + شريط «متبقٍ X دقائق».
- **FR-C12** ما بعد الطلب: تقييم 5 أبعاد + بقشيش، إبلاغ مشكلة، استرجاع بمتابعة، سجل بإعادة طلب مع إعادة فحص (C-53–C-58).
- **FR-C13** الحساب: سيارات، دفع، إشعارات داخلية، كوبونات، مفضلة، دعم بتذاكر، خصوصية (تنزيل بياناتي/حذف)، حذف حساب (C-59–C-69).
- **FR-C14** كل الحالات العامة الـ17 مبنية (قائمة 04).

## 2. ويب العميل

- **FR-W01** فتح مطعم وطلب كامل من رابط دون تطبيق (W-01–W-04).
- **FR-W02** متابعة الطلب من رابط + **Web Check-in** (أنا في الطريق/وصلت/الموقف) + رمز الاستلام (W-05–W-07).
- **FR-W03** لا خطوة تشترط تحميل التطبيق؛ دعوة ناعمة فقط.

## 3. شاشة تشغيل الفرع

- **FR-B01** دخول بكود فرع + PIN وربط جهاز مسمّى؛ ورديات فتح/إغلاق (B-01/B-02/B-16).
- **FR-B02** لوحة تشغيل حية بألوان أولوية وتنبيهات صوتية وملخص علوي (B-03).
- **FR-B03** قبول/قبول بتعديل وقت/رفض بسبب خلال العداد (B-04، BR-1).
- **FR-B04** KDS ثلاثي الأعمدة بمؤقتات ومنع «جاهز» ناقص (B-07/B-08).
- **FR-B05** السيارات في الطريق بترتيب أولوية الوصول + ETA + خريطة مصغرة مقيدة (B-09).
- **FR-B06** الواصلون: بطاقات كبيرة (سيارة، لوحة مختصرة، موقف، رمز) + خرج/وجدت/سلّمت/لم أجد + «الموظف متجه إليك» للعميل (B-13).
- **FR-B07** معالجة نقص منتج بموافقة العميل (B-06، BR-4).
- **FR-B08** مشكلات التسليم بتصعيد للمدير (B-14).
- **FR-B09** وضع الازدحام (B-15، BR-10).
- **FR-B10** يعمل PWA بملء الشاشة على تابلت/كاشير/متصفح، وقابل للتحويل لتطبيق Android لاحقاً.

## 4. بوابة التاجر

- **FR-M01** انضمام كامل بتحقق ومستندات ونسبة اكتمال (M-01، رحلة J13).
- **FR-M02** رئيسية بـKPIs ورسوم وتنبيهات (M-02).
- **FR-M03** علامة تجارية وفروع وإعدادات استلام + **معايرة النطاق بمحاكاة** (M-03–M-07).
- **FR-M04** قوائم: محرر يدوي كامل + مزامنة POS بحالة أخطاء وإعادة (M-08/M-09).
- **FR-M05** طلبات بفلاتر وخط زمني (M-10).
- **FR-M06** عروض وكوبونات بمشاركة تكلفة (M-11، BR-7).
- **FR-M07** موظفون وأدوار الجدول التسعة + مؤشرات غير عقابية (M-12).
- **FR-M08** عملاء بعرض مقنّع (M-13).
- **FR-M09** تقارير التشغيل/المبيعات/المشكلات بتصدير Excel/PDF (M-14).
- **FR-M10** مالية: تسويات، فواتير، نزاعات، حساب بنكي (M-15/M-16).
- **FR-M11** تكاملات بحالة واختبار اتصال (M-17).
- **FR-M12** سجل عمليات (M-18) واشتراك وفوترة (M-19، BR-14).

## 5. Super Admin

- **FR-A01** نظرة عامة بكل مؤشرات القائمة المعتمدة (A-01).
- **FR-A02** مركز عمليات مباشر بخريطة وتنبيهات قابلة للفتح والتواصل (A-02).
- **FR-A03** تجار: ملف كامل + أوامر (قبول، مستندات، تعليق، تفعيل، باقة، تجربة، رصيد، **دخول وضع الدعم مسجلاً**) (A-03) + رحلة انضمام بنسبة اكتمال (A-04).
- **FR-A04** فروع: نطاق، اختبار جيوفنس، جهاز، إعادة إرسال إعدادات (A-06).
- **FR-A05** طلبات بسجل غير قابل للتعديل (A-08، قاعدة 05§4-5).
- **FR-A06** عملاء ومستخدمون وموظفو منصة بالأدوار السبعة (A-07/A-21).
- **FR-A07** باقات واشتراكات + رسوم بسريان تاريخي (A-20، BR-14).
- **FR-A08** مدفوعات وتسويات واسترجاعات ونزاعات (A-09–A-11، وثيقة 13).
- **FR-A09** دعم بتذاكر مدمجة البيانات (A-15) ومخاطر بأدلة وقرار (A-16).
- **FR-A10** CMS كامل (A-13) يغذي التطبيق والموقع التعريفي.
- **FR-A11** صحة نظام + حوادث + Feature Flags + وظائف خلفية + Webhooks (A-23–A-26).
- **FR-A12** Audit Log لا يُحذف من الواجهة (A-22).
