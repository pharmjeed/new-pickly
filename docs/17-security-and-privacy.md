# 17 — الأمان والخصوصية (Security & Privacy)

المصدر السابق: 10، 13، 16 · الامتثال: PDPL من اليوم الأول

---

## 1. حماية الحساب

OTP محدود المحاولات + Rate Limiting · كشف الحسابات الوهمية (أجهزة/أنماط) · إلغاء الجلسات وDevice Sessions مسماة · تشفير Tokens (Secure Store في التطبيق) · CSRF للويب · Secure Cookies.

## 2. حماية التاجر

MFA للحسابات الحساسة · RBAC (وثيقة 16) · فصل الفروع · Audit Logs · إشعار تسجيل الدخول · تقييد الاسترجاع حسب الصلاحية والسقف · **تغيير الحساب البنكي = تحقق مزدوج + تنبيه مخاطر (A-16) + فترة تجميد payout**.

## 3. حماية الدفع

لا بطاقات خام في قاعدة البيانات (Tokenization، PCI Scope مخفض) · Signed Webhooks · Idempotency · Ledger مالي مستقل · Reconciliation يومي (وثيقة 13).

## 4. حماية الموقع والسيارة (جوهر خصوصية Pickly)

1. موافقة واضحة مسجلة (`user_consents`) قابلة للسحب.
2. الموقع **أثناء الطلب النشط فقط** — نص الشرح الحرفي في C-44، وإيقاف فوري عند الاكتمال/الإلغاء.
3. **تشفير رقم اللوحة** وعدم إظهاره كاملاً خارج الطلب التشغيلي (عمود مختصر للعرض).
4. بيانات السيارة لا تظهر لغير الفرع المعني وأثناء الطلب النشط فقط؛ في لوحة التاجر مقنّعة (M-13).
5. حذف/تقليل دقة بيانات الموقع بعد انتهاء الغرض: خام `pickup_location_events` يُحذف أو يُلخص بعد 30 يوماً من الإكمال.
6. سياسة احتفاظ محددة لكل صنف بيانات + سجل للموافقة والسحب.
7. حقوق العميل: تنزيل بياناته، طلب حذفها، حذف سجل السيارات (C-67)؛ حذف الحساب بعد تسوية المفتوح + OTP + فترة معالجة نظامية (C-69).
8. الفرع لا يرى رحلة العميل كاملة — ETA وخريطة مصغرة حول الفرع فقط.

## 5. حماية المنصة

WAF (Cloud Armor) · DDoS Protection · Secret Manager · تشفير أثناء النقل (TLS) والتخزين · نسخ احتياطية مجدولة ومختبرة الاستعادة · فحص Dependencies · SAST · Container Scanning · **اختبار اختراق قبل الإطلاق** (شرط في 20).

## 6. المخاطر والاحتيال (وحدة Risk)

إشارات مراقبة: تعدد حسابات، استردادات متكررة، محاولات دفع متكررة، إساءة كوبونات، طلبات مرتفعة غير معتادة، أجهزة/بطاقات مشتركة بين حسابات، اختلاف غير طبيعي بين موقع العميل والفرع، تسليم بلا رمز، تغيير بنكي للتاجر، مطعم يؤكد طلبات لم تُسلّم. كل تنبيه: درجة خطورة + سبب + أدلة + إجراءات مقترحة + قرار مراجع موثق.

## 7. حوكمة البيانات

تصنيف البيانات (شخصية/مالية/تشغيلية/موقعية) وسياسة وصول لكل صنف · بيئة Staging ببيانات اصطناعية لا حقيقية · وصول الدعم لبيانات العميل عبر «وضع الدعم» المسجل حصراً · Audit Log لا يُحذف من الواجهة.
