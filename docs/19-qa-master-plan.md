# 19 — خطة الجودة الرئيسية (QA Master Plan)

المصدر السابق: 05، 11، 13، 14

---

## 1. اختبارات Backend

Unit · Integration (قاعدة PostgreSQL اختبارية حقيقية) · **Payment Webhook Tests** (توقيع، تكرار، مبالغ خاطئة) · **Order State Machine Tests** (كل انتقال شرعي + رفض غير الشرعي) · **Tenant Isolation Tests** (محاولة قراءة/كتابة عبر الحدود = فشل إلزامي) · **Idempotency Tests** (تكرار كل POST مالي) · Refund Tests (جزئي، مكرر، سقوف) · Queue Retry + DLQ Tests.

## 2. اختبارات الويب (Playwright)

رحلة التاجر كاملة (انضمام → قائمة → طلب → تسوية) · تشغيل الفرع (قبول → KDS → طابور → تسليم) · Super Admin (أوامر حساسة + Audit) · الصلاحيات (كل دور يرى حدوده فقط) · RTL · Responsive · ويب العميل وWeb Check-in.

## 3. اختبارات التطبيق (أجهزة حقيقية iOS + Android)

صلاحيات الموقع (منح/رفض/سحب أثناء رحلة) · إغلاق التطبيق أثناء الرحلة (قيود Android للاستئناف / إحياء iOS بالجيوفنس) · ضعف الشبكة وانقطاعها · الرحلة في الخلفية · Push (وصول وفتح) · Deep Links (رابط طلب، رابط مطعم) · الدفع (3DS، Apple Pay، رفض) · **استعادة الطلب النشط بعد إعادة فتح التطبيق** · استهلاك البطارية في رحلة 30 دقيقة.

## 4. اختبارات الموقع الميدانية (إلزامية قبل الطيار)

داخل مدينة كثيفة · شارع سريع · **موقف متعدد الأدوار** · GPS غير دقيق · **الفرع داخل مول** · **فرعان قريبان** (لا وصول خاطئ) · المرور بجوار الفرع دون توقف (لا وصول كاذب — Dwell) · التوقف في إشارة قريبة · العميل يغير وجهته. لكل حالة: النتيجة المتوقعة موثقة ومقاسة (نسبة اكتشاف آلي، زر يدوي).

## 5. اختبارات التحمل

الأساس = حجم الإطلاق المتوقع، ويُختبر **بعشرة أضعافه**: تصفح متزامن · إنشاء طلبات · قبول الطلبات · تحديثات الموقع (أعلى حمل تكراري) · تنبيهات الوصول · WebSocket connections · حملات Push.

## 6. الأمان

فحص Dependencies + SAST + Container Scanning في CI · اختبار اختراق خارجي قبل الإطلاق · سيناريوهات: تجاوز عزل، تلاعب أسعار من العميل، إعادة webhook، تسليم بلا رمز، تصعيد صلاحيات.

## 7. الوصولية

WCAG AA على مسار الطلب والاستلام، قارئ شاشة على المسار الحرج، Dynamic Type.

## 8. بوابات الجودة في CI

لا دمج بدون: أخضر لكل الوحدات المتأثرة + Isolation + Idempotency + State Machine suites · تغطية العقود (contracts) 100% مولدة ومطابقة · E2E للـVertical Slice يعمل ليلياً · أي P0/P1 مفتوح يمنع الترقية للبيئة الأعلى.

## 9. بيئات

Local (docker-compose) → CI → **Staging مطابق للإنتاج** (بيانات تجريبية، مطاعم تجريبية، بوابة Sandbox، محاكاة طلبات، مراقبة كاملة) → Production. محاكي رحلة (سيارة افتراضية تتحرك على مسار) ضمن أدوات Staging لاختبار ETA والعتبات بلا قيادة فعلية.
