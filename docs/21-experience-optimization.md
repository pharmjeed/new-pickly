| البند | القيمة |

## القسم 1 — خريطة دمج الصفحات (Consolidation Map)

### تطبيق العميل: 69 شاشة منطقية ← **8 صفحات متطورة + 12 صفحة بسيطة**

| الصفحة المتطورة | تبتلع | الآلية |
|------------------|--------|--------|
| **P1 التهيئة** | C-01–C-04 | تسلسل واحد بخطوات |
| **P2 المصادقة** | C-05–C-08 | Stepper واحد: جوال ← OTP ← اسم؛ الإشعارات Sheet في سياقها |
| **P3 الاستكشاف الموحد** | C-09–C-13, C-16, C-17 | الرئيسية الأساس؛ البحث Overlay، التصنيفات Chips، الفلاتر Sheet، العروض تبويب |
| **P4 المطعم الشامل** | C-19–C-25 | المنيو داخل الصفحة؛ اختيار الفرع Sheet؛ المنتج+التخصيص Sheet واحد |
| **P5 السلة** | C-26, C-27, C-32 | تغير التوفر والكوبون Sheets |
| **P6 الإتمام** | C-28–C-31, C-33–C-37 | صفحة تمرير واحدة: وقت+سيارة+دفع+مراجعة؛ إضافة سيارة/بطاقة Sheets؛ النجاح حالة ختامية |
| **P7 الطلب الحي** ⭐ | C-38–C-52 | **صفحة واحدة تقودها آلة الحالات** — الهيكل ثابت والمحتوى يتبدل؛ الموقف/تأكيد السيارة/الرمز Sheets |
| **P8 الاكتمال وما بعده** | C-52–C-55 | التقييم والإبلاغ والاسترجاع Sheets فوق شاشة الاكتمال |
| صفحات بسيطة تبقى | C-14/15 (نتائج/خريطة)، C-18، C-56–C-69 | قوائم قياسية |

### شاشة الفرع: 16 ← **5 صفحات**
**لوحة التشغيل** (B-03) تبتلع B-04، B-05، B-06، B-09–B-14: التبويبات = الحالات، التفاصيل Drawer جانبي، القبول/الرفض/التسليم على البطاقة، نقص المنتج ومشكلات التسليم Sheets داخل الـDrawer. تبقى مستقلة: B-01 الدخول، B-02/B-16 الوردية (صفحة واحدة بوضعين)، **B-07/B-08 KDS** (شاشة مطبخ فعلياً مستقلة)، B-15 وضع الازدحام (Sheet لمدير الفرع).

### بوابة التاجر: 20 ← **14**
إعدادات الفرع صفحة بتبويبات (M-05+M-06+M-07 المعايرة تبويب) · المنيو والتوفر صفحة واحدة (M-08+M-09) · التسويات والفواتير صفحة بتبويبين (M-15+M-16).

### ويب العميل: 12 ← **4**
صفحة تتبع موحدة (W-05+W-06+W-07+W-08: التتبع + Check-in + الرمز + الفاتورة كحالات) + مطعم/سلة (W-01+W-02) + دخول/دفع (W-03+W-04) + طلباتي/دعم.

### ما لا يُدمج أبداً
مصادقة 3DS (صفحة البوابة)، KDS، لوحة الأدمن (أصلاً Modular Panel واحدة بوحدات)، الصفحات القانونية.

## القسم 2 — اختصارات الخطوات (Step Shortcuts)

| # | الاختصار | الأثر | القاعدة |
|---|----------|-------|---------|
| S1 | **Apple Pay السريع** من السلة | يتجاوز P6 كاملة للمستخدم العائد (السيارة الافتراضية + ASAP معبآن) | يظهر فقط عند وجود سيارة افتراضية |
| S2 | **إعادة الطلب بلمسة** من الرئيسية | «اطلب مرة أخرى» ← إتمام معبأ ← بصمة | إعادة فحص التوفر والأسعار إلزامية (C-58) |
| S3 | **إضافة سيارة مصغرة** | إلزامي حقلان فقط: اللون + آخر 4 أرقام | البقية تُستكمل لاحقاً من C-60 |
| S4 | **أزرار داخل الإشعار** | «أنا في الطريق» و«وصلت» Notification Actions دون فتح التطبيق | تُسجل كأحداث C-43/C-47 نفسها |
| S5 | **Live Activity / إشعار دائم** | تتبع P7 من شاشة القفل (iOS Live Activities / Android ongoing) | حالة الطلب + ETA + الرمز عند الوصول |
| S6 | **قبول تلقائي** للفروع الموثوقة | يلغي عداد الـ3 دقائق | شرط: 4 أسابيع تشغيل بنسبة قبول >98%؛ الرفض اليدوي يبقى متاحاً؛ إعداد لكل فرع |
| S7 | **جاهز تلقائياً من POS/KDS** | صفر لمسات بين القبول والتسليم | عند وجود تكامل فعال فقط |
| S8 | **تحقق بصري للطلبات العادية** | مطابقة السيارة + «سلّمت» + تأكيد العميل | الرمز/QR إلزامي ≥300 ر.س (BR-8) أو عند طلب العميل |
| S9 | موقف بـ QR (لوحة في كل موقف) | «وصلت + الموقف» بمسحة واحدة | اختياري للفروع المجهزة — v1.5 |

## القسم 3 — نطاق الطيار المشذّب (Pilot Scope Trim)

يُقسّم نطاق `01` إلى طبقتين. **الطيار (المرحلة 15) يبني الطبقة الأولى فقط:**

| يؤجل إلى ما بعد الطيار | البديل أثناء الطيار |
|--------------------------|----------------------|
| الطلب المجدول بفترات وسعات (C-28 جزئياً، BR-5) | ASAP فقط |
| ويب العميل الكامل (W-01–W-04، W-09–W-11) | التتبع + Web Check-in + الرمز فقط (الصفحة الموحدة) |
| نظام تذاكر الدعم الكامل (C-65/66، A-15) | زر واتساب/اتصال + سجل يدوي |
| الكوبونات بأنواعها (BR-7) | كود خصم نسبة واحد بسيط |
| البقشيش، خريطة الاكتشاف (C-15)، المفضلة (C-18، C-64) | — |
| من الأدمن: CMS الكامل، Feature Flags UI، محرك مخاطر آلي، العروض المشتركة | إعدادات ثابتة + مراجعة يدوية (الجداول تُنشأ، الواجهات تؤجل) |
| الظهور المدفوع والباقات المتعددة | باقة واحدة + فترة تجريبية |

**قاعدة:** المؤجل يُصمم له مكانه في ERD والعقود (لا إعادة بناء لاحقاً)، ويُوسم في فهرس الواجهات «مرحلة 2» دون بنائه.

## الأثر الإجمالي

- شاشات تُبنى للطيار: **~60 صفحة بدل 152** (بلا أي مساس بجوهر: اطلب ← قُد ← استلم من نافذتك).
- رحلة العائد: 3 لمسات من الرئيسية إلى طلب مدفوع (S1/S2).
- الطلب الحي: صفحة واحدة + شاشة القفل — لا تنقّل إطلاقاً أثناء القيادة.
