# 02 — الأطراف والأدوار (Personas & Roles)

المصدر السابق: 00، 01

---

## 1. الأطراف

**العميل** — يستعرض ويطلب ويدفع ويستلم من السيارة (أو زائر يتصفح فقط).
**التاجر (Merchant)** — الشركة/العلامة المالكة للمطعم أو المتجر. قد يملك عدة علامات (Brands).
**الفرع (Branch)** — الموقع الفعلي الذي يجهز ويسلّم.
**إدارة Pickly** — تشغّل المنصة.

## 2. أدوار جانب التاجر (9 أدوار)

| الدور | النطاق | ما يراه/يفعله |
|-------|--------|----------------|
| المالك (Owner) | كل العلامات والفروع | كل شيء + المالية + التكاملات + العقد |
| المدير العام | كل الفروع | كالمالك عدا الحساب البنكي والعقد |
| مدير العمليات | فروع معيّنة أو الكل | تشغيل، قوائم، تقارير تشغيلية |
| مدير الفرع | فرعه | كامل تشغيل الفرع: وضع الازدحام، الورديات، موظفو فرعه، تصعيدات التسليم |
| الكاشير | فرعه | الطلبات: قبول/رفض/تعديل حالة، معالجة نقص منتج |
| المطبخ/التجهيز | فرعه | شاشة KDS: بدء تحضير، غير متوفر، جاهز |
| موظف التسليم | فرعه | السيارات الواصلة والطلبات الجاهزة، خرج/وجدت/سلّمت، رمز التحقق |
| المالية (المحاسب) | التاجر | التسويات والفواتير فقط |
| محلل تقارير | التاجر | تقارير قراءة فقط |

مشرف الطلبات = تكوين من صلاحيات الكاشير + رؤية كل ألواح الفرع (دور مركّب، لا جدول مستقل).

## 3. أدوار جانب Pickly (7 أدوار)

| الدور | الصلاحية |
|-------|----------|
| Super Admin | كل شيء، بما فيه تجاوز العزل (يُسجل في Audit Log) وFeature Flags والإعدادات |
| Operations | مركز العمليات، الفروع، الطلبات، التدخل التشغيلي |
| Finance | المدفوعات، التسويات، الاسترجاعات، الفواتير، الباقات |
| Support | التذاكر، ملفات الطلبات قراءة، تنفيذ استرجاع ضمن سقف |
| Merchant Success | طلبات الانضمام، مراجعة التجار، التفعيل، المحتوى التجاري |
| Risk | تنبيهات الاحتيال، الحظر، مراجعة الأدلة والقرار |
| Read Only (المدقق) | قراءة كل شيء، تعديل لا شيء |

المحتوى والتسويق تعمل عبر وحدة CMS بصلاحية يمنحها Super Admin؛ الإدارة التقنية عبر صحة النظام وFeature Flags وWebhooks.

## 4. شخصيات تصميمية (Personas) موجزة

- **أبو فهد (عميل عائلي):** يطلب من السيارة وأطفاله معه — لا يريد النزول إطلاقاً؛ يحتاج وضوح «متى أتحرك» ورمز استلام بسيطاً.
- **سارة (موظفة وقت الغداء):** 35 دقيقة استراحة — تطلب قبل التحرك، ETA دقيق أهم شيء لديها.
- **زائر بلا تطبيق:** وصله رابط — يكمل الرحلة من ويب العميل دون تحميل.
- **عبدالله (كاشير فرع مزدحم):** شاشة واحدة، أزرار كبيرة، صوت للوصول — لا وقت للتنقل بين قوائم.
- **مشغل المطبخ:** يرى KDS فقط: جديد/قيد التحضير/جاهز مع مؤقتات.
- **أم خالد (مالكة سلسلة 12 فرعاً):** تقارير مقارنة للفروع، تسويات واضحة، ولا تريد تدريباً معقداً لموظفيها.
- **مشغل عمليات Pickly:** خريطة حية، تنبيهات فروع غير متصلة وسيارات متكدسة، تدخل سريع.

## 5. قواعد عامة

1. الأدوار في هذين الجدولين **قائمة مغلقة** — أي دور جديد يتطلب تعديل هذه الوثيقة ثم `16-rbac-matrix.md`.
2. بيانات السيارة واللوحة تظهر كاملة **فقط** لموظفي الفرع المعني أثناء الطلب النشط.
3. لا عقوبات آلية مبنية على مؤشرات الموظفين — مؤشرات تشغيلية يراجعها المدير.
