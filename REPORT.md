# التقرير النهائي — تغطية Pickly مقابل الوثائق الحاكمة

> آخر تحديث: 2026-07-09 · المستودع: https://github.com/pharmjeed/new-pickly

## 1. الخلاصة التنفيذية

المنتج يعمل **E2E كاملاً محلياً بالمحاكيات** بأمر واحد، ورحلة J1 (اطلب ← قُد ← استلم من نافذتك)
خضراء ببوابتي اختبار آليتين (تكاملي 11 اختباراً + Playwright في المتصفح بواجهتي العميل والفرع)،
وتحت حمل 200 طلب/دقيقة بصفر أخطاء. كل اعتماد خارجي خلف Adapter بوضعي mock/production —
التفعيل الإنتاجي = لصق مفاتيح في Secret Manager (قائمة HUMAN-ACTIONS.md).

## 5. أدلة الجودة (كلها قابلة لإعادة التشغيل)

- `pnpm test` — 33 اختباراً (عقود + وحدات + تكاملي J1 + suites إلزامية).
- `pnpm --filter @pickly/e2e test` — رحلة J1 في المتصفح (عميل+فرع) ≈ 35 ثانية.
- k6: 200 طلب/دقيقة × دقيقتين = 401 رحلة، 0/4411 فشل، p95=89ms (الهدف <800ms).
- CI (GitHub Actions): lint + typecheck + هجرات + tests + ركن e2e كامل.

## 6. فجوات معلومة (بأولوية ما بعد الطيار)

1. تطبيق Expo: push الخلفي وLive Activity وgeofence الخلفي تتطلب Dev Build وحسابات B5.
2. الواجهات تستخدم polling (2.5s) — قناة WS جاهزة بالخادم؛ ربط الواجهات بها تحسين تدريجي.
3. حمل 2000 اتصال WS — يُقاس بعد ربط الواجهات بالقناة.
4. كوبونات/مجدول/محفظة/ولاء — جداولها وأعلامها جاهزة، الواجهات «مرحلة 2» بقرار docs/21§3.
5. صفحات merchant-web المتبقية عن الـ14 (إعدادات الفرع M-05 بمعايرة M-07، الفواتير M-16) — الـAPI للمعايرة موجود (branch_pickup_settings).
